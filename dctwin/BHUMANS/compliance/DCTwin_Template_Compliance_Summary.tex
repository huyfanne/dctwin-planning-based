\section*{Compliance verdict}
DCTwin adopts both dcwiz standard policy templates. RecommendTemplate is subclassed by WeeklyPlanTemplate (src/\allowbreak{}plan\_\allowbreak{}weekly.py:22, import :11); TrajectoryPolicyTemplate is subclassed by AITrajectoryReplay (src/\allowbreak{}ai\_\allowbreak{}trajectory\_\allowbreak{}test.py:14, import :8) and BaselineTrajectory (src/\allowbreak{}baseline\_\allowbreak{}policy\_\allowbreak{}test.py:12, import :6). The base-class lifecycle \_\allowbreak{}\_\allowbreak{}init\_\allowbreak{}\_\allowbreak{} $\rightarrow$ \_\allowbreak{}\_\allowbreak{}call\_\allowbreak{}\_\allowbreak{} $\rightarrow$ initialize $\rightarrow$ run is honored: subclasses override initialize() (and run() for the planner) at the designated extension points (plan\_\allowbreak{}weekly.py:30,67). The sample\_\allowbreak{}template/\allowbreak{} layout for configs/\allowbreak{}, data/\allowbreak{}, and models/\allowbreak{} is mirrored: device-map JSON schemas and the dt/\allowbreak{}test/\allowbreak{}train prototxt shapes match identically or near-identically. DCTwin intentionally omits the RL-training artifacts (ai\_\allowbreak{}policy\_\allowbreak{}train.py, single-file hooks.py, policy/\allowbreak{}policy.pth, recommend.prototxt, train\_\allowbreak{}test.prototxt) by design, because it is a search/\allowbreak{}planning optimizer, not a trained dcbrain RL policy; those responsibilities are relocated into planner/\allowbreak{}. One honest defect remains to remediate: AITrajectoryReplay calls run(policy="ai") without setting self.test\_\allowbreak{}collector, so the template's run\_\allowbreak{}ai() raises at runtime.

\section*{Evidence by template}
DCTwin subclasses both dcwiz standard policy templates and mirrors the sample\_\allowbreak{}template layout for configs/\allowbreak{}, data/\allowbreak{}, and models/\allowbreak{}. Each base-class contract (\_\allowbreak{}\_\allowbreak{}init\_\allowbreak{}\_\allowbreak{} $\rightarrow$ \_\allowbreak{}\_\allowbreak{}call\_\allowbreak{}\_\allowbreak{} $\rightarrow$ initialize $\rightarrow$ run) is honored at the designated extension points. All DCTwin-side (src/\allowbreak{}...) citations below were re-verified against the repo. Template-side citations are taken as given from the source report (external dcwiz/\allowbreak{}dcbrain sample). Verdict vocabulary: followed = contract honored as-is; adapted = honored with justified divergences; partial = honored with a gap; missing = absent by design; extended = honored plus domain additions.

\smallskip\noindent\textbf{(a) RecommendTemplate $\rightarrow$ WeeklyPlanTemplate [adapted]}\\[1pt]
WeeklyPlanTemplate subclasses RecommendTemplate and imports it from dcwiz\_\allowbreak{}policy\_\allowbreak{}template, overriding initialize() to wire forecaster/\allowbreak{}oracle/\allowbreak{}beam-planner and fully replacing run() to call run\_\allowbreak{}weekly\_\allowbreak{}plan(). The base run() (which expects a reactive dcbrain policy and would call log\_\allowbreak{}recommendations()) is never reached -- a justified divergence for a deterministic batch planner, not a contract violation.

\section*{Why DCTwin diverges (by design)}
One architectural fact explains every "missing" verdict: DCTwin is a search/\allowbreak{}planning optimizer, not a trained RL agent. The dcwiz template assumes a dcbrain SAC policy persisted as policy/\allowbreak{}policy.pth, fed per-step rewards and states through hooks.py, and trained by ai\_\allowbreak{}policy\_\allowbreak{}train.py. DCTwin instead beam-searches three global setpoints (SAT, airflow, CHWST), scoring every candidate with a full-week EnergyPlus oracle run. That swap removes the RL training pipeline and relocates its responsibilities into the planner/\allowbreak{} package and JSON config, verified present in src/\allowbreak{} (all targets below confirmed).

\section*{One item to remediate}
One honest defect. AITrajectoryReplay (src/\allowbreak{}ai\_\allowbreak{}trajectory\_\allowbreak{}test.py:14) overrides initialize() at :17-28 to set self.env and self.act, but never sets self.test\_\allowbreak{}collector, then is invoked with policy="ai" at :36. The template's run\_\allowbreak{}ai() requires a test collector and raises ValueError("Test collector is not set"), so the call crashes as written. Fix, two options: set self.test\_\allowbreak{}collector in initialize(), OR call with policy="baseline" (the fixed-action replay is baseline-shaped anyway). BaselineTrajectory (src/\allowbreak{}baseline\_\allowbreak{}policy\_\allowbreak{}test.py:12) is unaffected and fully followed.


\section*{Conclusion}
\begin{itemize}
\item RecommendTemplate: followed via adaptation. WeeklyPlanTemplate (plan\_\allowbreak{}weekly.py:22) imports from dcwiz\_\allowbreak{}policy\_\allowbreak{}template (:11), overrides initialize() (:30) and run() (:67) at the designated extension points, and honors the \_\allowbreak{}\_\allowbreak{}init\_\allowbreak{}\_\allowbreak{}/\allowbreak{}\_\allowbreak{}\_\allowbreak{}call\_\allowbreak{}\_\allowbreak{} lifecycle.
\item TrajectoryPolicyTemplate: followed for baseline, partial for AI replay. BaselineTrajectory honors the contract end-to-end via policy="baseline"; AITrajectoryReplay honors structure + initialize() but has the test\_\allowbreak{}collector gap above.
\item configs/\allowbreak{}, data/\allowbreak{}, models/\allowbreak{}: honored with extensions. JSON device-map schemas, EPW weather, schedule-folder convention, and dt/\allowbreak{}test/\allowbreak{}train prototxt shapes all match; DCTwin adds calibration/\allowbreak{}realized-history, pumps/\allowbreak{}, and forecaster.pkl without breaking shared schemas.
\item RL-training artifacts (ai\_\allowbreak{}policy\_\allowbreak{}train.py, single-file hooks.py, policy/\allowbreak{}policy.pth, recommend.prototxt, train\_\allowbreak{}test.prototxt): intentionally absent, replaced by fit\_\allowbreak{}forecaster.py, the planner/\allowbreak{} package, and BeamPlanner.
\item Net: the template contracts (base-class subclassing, the initialize/\allowbreak{}run/\allowbreak{}\_\allowbreak{}\_\allowbreak{}call\_\allowbreak{}\_\allowbreak{} lifecycle, the config/\allowbreak{}data/\allowbreak{}model layouts) are honored. One gap to fix: the AITrajectoryReplay test\_\allowbreak{}collector omission; everything else is a justified architectural adaptation.
\end{itemize}
